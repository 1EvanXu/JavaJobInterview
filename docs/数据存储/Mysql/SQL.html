SQL优化

优化查询？

  1. 业务层：业务拆分（查看是否有字段冗余），前后拆分（根据时间分表）
  2. 数据库层：读写分离，主从分离
  3. SQL层：
    1. 访问的数据过多或者访问了不需要的记录
    2. 大查询分解成多个小查询，而且分解后能让缓存效率更高和减少锁的竞争
    3. 合理使用索引：是否建立聚集索引，使用B+索引还是散列索引
    4. 避免复杂的正则表达式
    5. 避免或者简化排序
